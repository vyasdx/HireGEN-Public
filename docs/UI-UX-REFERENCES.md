## 1. Zoom Homepage Teardown

**Reference:** https://www.zoom.com/  
**Reviewed:** 2026-05-25  
**Why this reference matters:** Enterprise SaaS homepage with broad product suite, AI positioning, clear CTAs, and strong product-card storytelling.

---

## 1.1 Overall Impression

Zoom's homepage feels like a mature enterprise platform rather than a single-product landing page.

The visual system communicates:

- Trust
- Platform breadth
- AI-first positioning
- Enterprise readiness
- Consumer familiarity

For HireGEN, the useful pattern is:

> One strong central promise, then multiple product surfaces shown as tangible cards.

---

## 1.2 Page Structure

Zoom's page roughly follows this structure:

1. Global navigation
2. Announcement banner
3. Large centered hero
4. Primary and secondary CTA buttons
5. Horizontal product card carousel
6. Use-case / solution sections
7. Customer trust logos and ratings
8. Customer stories
9. News / updates
10. Final CTA
11. Dense enterprise footer

---

## 1.3 What Works Well

### Navigation

Zoom uses a dark, high-contrast navigation bar with:

- Logo on the left
- Product and solution dropdowns
- Pricing link
- Search
- Meet dropdown
- Sign In
- Support
- Contact Sales
- Sign Up Free
- App grid icon

Why it works:

- Enterprise users can find sales/support quickly.
- Free signup remains prominent.
- Product breadth is hidden behind dropdowns instead of crowding the hero.

HireGEN adaptation:

- Keep nav simpler for MVP.
- Suggested nav:
  - Product
  - Career Guide
  - Validation
  - Recruiters
  - Pricing
  - Sign in
  - Join beta

### Announcement Banner

Zoom uses a slim banner promoting "AI note taking across platforms" with a gradient CTA.

Why it works:

- Promotes a timely feature without changing the main hero.
- Creates a sense of freshness.

HireGEN adaptation:

- Banner idea:
  - "OpenAI x Outskill Hackathon build in progress"
  - "Join the HireGEN proof-profile beta"
  - "New: Git Verified and Lab Proof badges"

### Hero Section

Zoom's hero is centered, large, simple, and non-technical:

> Find out what's possible when work connects

The subtext explains the platform:

> Bridge the gap between talking and doing with the AI-first work platform built for you.

Why it works:

- The headline is emotional.
- The subcopy makes it business-relevant.
- CTA pair gives two paths: explore products / find plan.

HireGEN adaptation:

Potential hero:

> Get hired on proof, not polish.

Subcopy:

> HireGEN turns resumes, GitHub portfolios, projects, and validation tasks into structured skill graphs recruiters can trust.

CTA pair:

- Build proof profile
- View recruiter demo

### Product Card Carousel

Zoom's hero flows into a horizontal product carousel showing cards like:

- BrightHire
- Virtual Agent
- Meetings
- Contact Center
- Phone
- My Notes

Why it works:

- Shows platform breadth visually.
- Each product card has a distinct use case.
- It creates momentum below the hero.

HireGEN adaptation:

Use product cards:

- Proof Profile
- Skill Graph
- Git Verified
- Lab Proof
- Career Guide
- Recruiter Dashboard
- AI Job Impact Explorer

This is especially useful for HireGEN because our product has multiple modules but we should avoid explaining all of them in text.

### Trust Section

Zoom uses:

- Customer logos
- Review ratings
- Customer stories

Why it works:

- Makes the platform feel safe and established.

HireGEN adaptation:

Since HireGEN is pre-MVP, do not fake credibility.

Use instead:

- "Built for Indian tech hiring"
- "Hackathon MVP"
- "Proof signals we validate"
- Badge system preview
- Transparent disclaimers: "Pilot data coming soon"

### Final CTA

Zoom repeats the conversion moment near the bottom:

- Get started today
- Find your plan

HireGEN adaptation:

- Create proof profile
- Join recruiter pilot
- View demo candidate

## 1.4 What HireGEN Should Avoid

Do not copy Zoom's full complexity.

Avoid:

- Too many nav items before product maturity
- Too many product modules in the first viewport
- Generic "AI-first platform" language
- Fake enterprise logos
- Overly broad audience messaging

HireGEN needs sharper specificity:

> Indian candidates. Verified proof. Recruiters trust the signal.

---

## 1.5 Recommended HireGEN Homepage Direction

### First Viewport

1. Dark/navy header
2. Hackathon or beta announcement banner
3. Big centered hero
4. Two CTAs
5. Product-card rail peeking into first fold

### Product Rail Cards

Suggested cards:

| Card | Purpose |
|---|---|
| Proof Profile | Candidate identity built from evidence |
| Git Verified | GitHub portfolio verification |
| Lab Proof | Controlled validation task |
| Skill Graph | Structured skill map |
| Career Guide | Resume-to-roadmap analysis |
| Recruiter Dashboard | Verified shortlist view |

### Visual Style

Recommended:

- Deep navy header
- Blue/indigo hero gradient
- White product cards
- Real UI mockups inside cards
- Badge icons as proof signals
- Strong button contrast

Avoid:

- Too much glassmorphism during hackathon
- Decorative gradients without product meaning
- Overcrowded text blocks

---

## 1.6 Direct UX Takeaways for Hackathon MVP

Use Zoom's page pattern to make HireGEN look bigger than the MVP while staying honest:

1. Hero sells the main promise.
2. Product rail shows modules.
3. Demo flow proves one module deeply.
4. Badge system creates visual trust.
5. Recruiter view shows business value.

Recommended hackathon homepage order:

1. Nav
2. Hackathon banner
3. Hero: "Get hired on proof, not polish."
4. CTA: "Build proof profile" / "View recruiter demo"
5. Product rail: Proof Profile, Git Verified, Lab Proof, Skill Graph, Career Guide, Recruiter Dashboard
6. Live demo section
7. How skill graph works
8. Candidate badge system
9. Recruiter proof view
10. Final CTA
